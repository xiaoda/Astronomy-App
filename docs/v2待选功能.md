# v2待选功能

我看了当前 v1 的实现，核心已经很明确了：`星空背景 + 长按流星 + 文案轮播 + 背景音乐 + 基础设置`。它更像一个“沉浸式陪伴 App”，不是工具型天文产品。基于这个现状，我建议下一步不要急着做重功能，先把“可感知价值”和“可留存体验”补起来。

## 先定方向

我会优先推荐走“沉浸陪伴型”，这是和你现在完成度最连续的一条路。

- `沉浸陪伴型`：继续做氛围、情绪、陪伴感。最容易把 v1 做精致，改动也最顺。
- `轻探索型`：加入少量星座、月相、今晚可见天象。会更像“有内容的星空 App”。
- `工具型`：做观测、定位、星图、提醒。这条最重，也最容易把现在的气质做散。

## 最值得先加的

- `首次引导`：现在“长按 1 秒触发流星”有创意，但发现成本偏高。建议首次进入时做 2 到 3 条轻提示，几秒后自动消失。
- `设置记忆`：把“显示文案 / 背景音乐 / 是否已看过引导”持久化。这个很小，但体感提升非常直接。
- `音乐控制补齐`：加当前曲目名、下一首、音量、淡入淡出。现在只有开关，用户对音乐缺少掌控感。
- `文案能力升级`：支持“换一句”“收藏一句”“按情绪切换文案包”，比如安静、鼓励、睡前。
- `专注/陪伴模式`：做一个 5 分钟、10 分钟、20 分钟的沉浸计时，开始后自动隐藏多余 UI，只保留星空和文案。
- `低干扰优化`：增加“低动态模式”或“省电模式”，照顾容易晕动、设备较弱、睡前使用的场景。

## 适合做成 v2 亮点的

- `今夜主题`：深夜、月夜、极光、流星雨几个主题切换，比单纯加设置项更有记忆点。
- `时间联动`：根据本地时间自动调整星空亮度、色温、文案气质，晚上更沉静，凌晨更克制。
- `轻天文信息`：不要一上来做复杂星图，只加“今日月相”“今晚一句天象提示”“一颗可点亮的星座热点”就够了。
- `分享卡片`：把当前星空 + 一句文案 + 时间，导出成一张分享图。这类产品很适合做传播入口。
- `声音场景`：除了纯音乐，再加白噪音、宇宙环境音、雨声混合，会更贴近陪伴/助眠使用场景。

## 我反而不建议现在就做的

- `重知识库`：星表、百科、复杂天文解释，投入大，而且会冲淡现在的产品气质。
- `复杂账号体系`：v1.1 阶段没必要。
- `过多设置项`：你现在的优点是轻，别太快做成“参数面板”。

如果只选 3 个我会先做：`首次引导`、`设置记忆`、`音乐控制补齐`。这三项投入小，但最能立刻改善“第一次用”和“愿不愿意多待一会儿”的体验。

如果你愿意，下一轮可以继续把这些建议收敛成一版 `v1.1 功能优先级清单`。
