# 文档目录规范

本目录用于集中管理产品规划类文档，按“同一时期/版本一个文件夹”归档，避免同类文档跨目录分散。

## 目录结构

- `docs/YYYY-MM-DD-vX-主题/`

每个时期目录下固定放三类文档：

- `功能清单.md`
- `开发计划.md`
- `任务看板.md`

示例：

- `docs/2026-02-27-v1-MVP-无声音版/功能清单.md`
- `docs/2026-02-27-v1-MVP-无声音版/开发计划.md`
- `docs/2026-02-27-v1-MVP-无声音版/任务看板.md`

## 命名规则

- 时期目录命名：`YYYY-MM-DD-vX-主题`
- 文件命名固定使用：`功能清单.md`、`开发计划.md`、`任务看板.md`

## 版本管理建议

- 同一主题迭代时，`vX` 递增（`v1`、`v2`、`v3`...）。
- 日期使用该批文档创建或主要修订日期。
- 不覆盖旧版本，新增时期目录保留历史追溯能力。
- 同一时期尽量保持三份文档齐全，便于查看和对比。
