# scenes 总览

## 目标
- `scenes/` 用来承载差异较大的沉浸式场景。
- 每个场景可以拥有自己的背景、交互、文案、音效与样式。
- 新增场景时，优先新建独立目录，而不是继续把逻辑堆进 `App.tsx`。

## 当前结构
- `registry.ts`: 统一维护场景元数据与路径。
- `home/`: 当前已接入的首页沉浸场景。

## 推荐边界
- `app/` 只负责路由和应用壳。
- `pages/scene-hub/` 只负责场景选择入口。
- `scenes/<scene>/` 负责该场景自己的表现与逻辑。
- 只有明确跨场景复用的东西，才考虑放进 `shared/` 或保留在 `features/`
